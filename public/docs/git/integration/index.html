Регистрация#

Вероятно, сейчас или в будущем вы захотите разрабатывать проекты совместно с другими разработчиками. Для этого вам потребуется зарегистрироваться в GitHub. Напоминаем, что GitHub — это облачная платформа для хостинга Git-репозиториев.

В подавляющем большинстве случаев разработчики выкладывают свои open-source проекты именно на GitHub, но есть и другие сервисы. Например, GitLab, Bitbucket, Azure DevOps.

SSH-ключи#

Когда вы связываетесь с серверами GitHub через командную строку, вы используете один из двух протоколов безопасности: SSH и HTTPS. HTTPS проще в настройке, но SSH обеспечивает более высокий уровень безопасности. В этой инструкции мы воспользуемся SSH.

С помощью SSH-ключа вы сможете подключаться к GitHub без необходимости каждый раз вводить данные для аутентификации.

Создание ключа#

  1. Сгенерируйте SSH-ключ в вашей системе

Когда появится запрос Enter a file in which to save the key, нажмите Enter. Ключ появится в директории по умолчанию ~/.ssh/

Когда появится запрос Enter passphrase (empty for no passphrase), придумайте и введите пароль для SSH-ключа.

  1. Активируйте SSH-агент и добавьте в него SSH-ключ

SSH-агент — это программа, которая хранит ваши SSH-ключи и предоставляет их GitHub по запросу.

Добавление SSH-ключа в учетную запись GitHub#

  1. Скопируйте ключ в буфер обмена.
  1. В вашем аккаунте GitHub откройте настройки и перейдите на вкладку SSH and GPG keys.
  2. Нажмите кнопку New SSH key.
  3. Напишите любое название в поле Title и вставьте из буфера обмена SSH-ключ в поле Key.
  4. В поле Key type выберите Authentication Key .
  5. Нажмите Add SSH key.

Создание удаленного репозитория#

  1. Перейдите на страницу с вашими проектами и нажмите кнопку New.
  2. Введите имя проекта в поле Repository name.
  3. (опционально) Введите описание проекта в поле Description.
  4. Выберите тип проекта. Публичный виден всем, приватный — только вам.
  5. (опционально) Создайте дополнительные файлы:
    • README.md — описание проекта. Мы его уже создали, пропустите этот шаг.
    • .gitignore — список игнорируемых файлов. Не всегда имеет смысл загружать на GitHub все файлы из локального репозитория.
    • Лицензия — позволяет или запрещает другим использовать ваш код.
  6. Нажмите кнопку Create repository.

Связывание локального и удаленного репозиториев#

Теперь у вас есть два репозитория: в вашей системе и на GitHub. Осталось их связать, чтобы в дальнейшем изменения синхронизировались.

Обратите внимание

Команды выполняются из директории проекта.

  1. Свяжите репозитории командой git remote add, которая принимает два аргумента:

    • Название удаленного репозитория, как правило, origin;
    • URL-адрес удаленного репозитория.

  2. (опционально) Переименуйте основную ветку. По умолчанию основная ветка в Git называется master, а в GitHub – main. Неважно, какое название вы выберете, главное, чтобы оно совпадало.

  3. Запушьте (отправьте) ваш локальный репозиторий на сервер. Параметр -u используется только первый раз, он связывает ветки. В дальнейшем достаточно писать git push.

    Если вы увидите следующее сообщение, то введите yes.

Готово. Можете обновить страницу и изучить структуру вашего нового репозитория на GitHub.
