Отправка изменений#

Локальные ветки не синхронизируются автоматически с удаленными. Если вы сейчас используете команду git push, то отправите в удаленный репозиторий только ветку main, которую связали ранее.

Отправьте в удаленный репозиторий ветку feature. Правда, вы ее недавно удалили, так что создайте заново.

А теперь введите команду git branch -vv, которая покажет локальные ветки и связанные с ней удаленные.

Обратите внимание на запись origin/main. Это — ветка слежения, то есть локальная ветка, которая передает информацию об удаленной. Если другой разработчик внесет изменения в эту ветку на GitHub, вы сможете узнать об этом с помощью ветки слежения и скопировать себе актуальную версию.

Однако напротив ветки feature такая запись отсутствует, потому что мы отправили ее в репозиторий, но не настроили на слежение. Удостоверьтесь, что находитесь в нужной ветке и выполните команду git branch -u <удаленный_репозиторий>/<имя_ветки>.

Но проще делать это еще на этапе git push, добавив параметр -u.

Повторите команду git branch -vv и убедитесь, что ветка слежения origin/feature появилась.

Важно!

Информация из git branch -vv может быть устаревшей. Эта команда не обращается к серверам, а использует локальный кэш. Кэш обновляется только при командах git push, git pull и git fetch.

Получение изменений#

В совместных проектах вам часто придется обновлять свою локальную копию репозитория на основе изменений от других разработчиков. Это можно сделать двумя способами.

git fetch#

Команда git fetch <удаленный_репозиторий> загружает из удаленного репозитория все изменения, но не заменяет автоматически локальные файлы. Это удобно, так как вы можете сначала изучить эти изменения, а уже потом применить их командой git merge <удаленный_репозиторий>/<имя_ветки>.

git pull#

Команда git pull <удаленный_репозиторий> <имя_ветки> совмещает git fetch и git merge. Все изменения применяются автоматически.

Удаление изменений#

Чтобы удалить ветку в в удаленном репозитории, используйте команду git push <удаленный_репозиторий> --delete <имя_ветки>.
